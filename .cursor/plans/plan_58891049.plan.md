## Final shape (consolidated from discussion)

- **Pattern:** event bus (`EventTarget` singleton).
- **Foundation:** custom (Ōēł150 LOC, zero deps) ŌĆö keeps every backend swap a one-file change.
- **Backend:** `@sentry/astro` hosted, with `@spotlightjs/astro` for local dashboard testing without an account.
- **Activation:** dev = always-on; prod = opt-in via `?debug=1`, `localStorage.debug='1'`, or `Ctrl+Shift+D`.
- **Signals:** namespaced logs, JS errors + unhandledrejection, live state, FPS / Web Vitals, network requests.

## Architecture

```mermaid
flowchart TD
    callSites["debug('demo:rob').info(...)<br/>debug('theme').error(...)"]
    callSites --> bus["lib/debug.ts<br/>(EventTarget bus +<br/>ring buffer + level gate)"]

    bus --> consoleMirror["console mirror<br/>(dev only)"]
    bus --> overlay["DebugOverlay.tsx<br/>Logs / State / Perf / Network"]
    bus --> sentryTransport["lib/debug-sentry.ts<br/>captureException +<br/>addBreadcrumb"]

    netTap["lib/debug-network.ts<br/>fetch + XHR taps"] --> bus
    fpsTap["FPS / Web Vitals sampler"] --> bus
    errTap["window.onerror +<br/>unhandledrejection"] --> bus

    sentryTransport --> sentrySdk["@sentry/astro SDK"]
    sentrySdk --> spotlight["@spotlightjs/astro<br/>(local dashboard, dev)"]
    sentrySdk --> sentryCloud["sentry.io<br/>(once DSN added)"]
```

The bus is the single producer surface. Subscribers are added/removed in one
file each. Spotlight intercepts the same SDK events locally ŌåÆ no account
needed to evaluate.

## Files to create

1. `[PersonalPortfolio/src/lib/debug.ts](PersonalPortfolio/src/lib/debug.ts)` ŌĆö singleton bus.
   - `debug(namespace)` returns `{ trace, info, warn, error }`.
   - Ring buffer (last 500 entries) + per-level filtering.
   - Emits `CustomEvent('debug:log' | 'debug:network' | 'debug:perf' | 'debug:error')` on `window`.
   - Reads enabled state from `window.__DEBUG_ENABLED`.
   - In `import.meta.env.DEV`, always mirrors to `console.*`.

2. `[PersonalPortfolio/src/lib/useDebug.ts](PersonalPortfolio/src/lib/useDebug.ts)` ŌĆö `useMemo`-wrapped hook returning `debug(ns)` for `.tsx` demos.

3. `[PersonalPortfolio/src/lib/debug-network.ts](PersonalPortfolio/src/lib/debug-network.ts)` ŌĆö wraps `window.fetch` + `XMLHttpRequest.prototype.open/send` to emit `'debug:network'` events with `{method, url, status, ok, durationMs, startedAt}`. Installed only when debug is enabled.

4. `[PersonalPortfolio/src/lib/debug-sentry.ts](PersonalPortfolio/src/lib/debug-sentry.ts)` ŌĆö bus subscriber that maps:
   - `info` ŌåÆ `Sentry.addBreadcrumb({ level: 'info', ... })`
   - `warn` ŌåÆ `Sentry.addBreadcrumb({ level: 'warning', ... })`
   - `error` ŌåÆ `Sentry.captureException(err)` (or `captureMessage` if no Error)
   - network events ŌåÆ `Sentry.addBreadcrumb({ category: 'fetch', ... })`

5. `[PersonalPortfolio/src/components/DebugInit.astro](PersonalPortfolio/src/components/DebugInit.astro)` ŌĆö `<script is:inline>` that:
   - Computes `__DEBUG_ENABLED` from `URLSearchParams.debug` / `localStorage.debug` (mirrors pattern from `[PersonalPortfolio/src/components/ThemeInit.astro](PersonalPortfolio/src/components/ThemeInit.astro)` lines 33ŌĆō50).
   - Always-on in `import.meta.env.DEV`.
   - Installs `window.addEventListener('error' | 'unhandledrejection', ...)` with a pre-init queue drained once `lib/debug.ts` loads.
   - Registers `Ctrl+Shift+D` toggle.
   - Lazy-loads overlay + network tap + Sentry subscriber on first activation.

6. `[PersonalPortfolio/src/components/DebugOverlay.tsx](PersonalPortfolio/src/components/DebugOverlay.tsx)` ŌĆö React island (`client:idle`) with tabs:
